Introduction

The SSC provides a synchronous communication link with external devices. It supports many serial synchronous communication protocols. The SSC USART mode driver emulates an USART over the SSC: it allows to connect the AVR32 to an USART through the SSC controller.

 

SSC USART mode Driver

The SSC USART mode driver philosophy of usage is:
  • Initializes an SSC for use as USART, by setting appropriate registers [ssc_uart_init]
  • Writes a character to SSC UART / reads a character from SSC UART [ssc_uart_putchar / ssc_uart_getchar]
  • Returns whether or not a parity error has occured [ssc_uart_parity_error]
  • Returns whether or not a framing error has occured [ssc_uart_framing_error]
  • Returns whether or not a overrun error has occured [ssc_uart_overrun_error]
  • Resets the error flags of the SSC UART [ssc_uart_reset_status].

     

    This driver is composed of ssc_usart.c and ssc_usart.h.

     

    An example is available: Echo character typed on an hyperterminal window through a USART communication link.
